\SS_ClassManifest

A utility class which builds a manifest of all classes, interfaces and some additional items present in a directory, and caches it.

It finds the following information:
- Class and interface names and paths.
- All direct and indirect descendants of a class.
- All implementors of an interface.
- All module configuration files.

    Methods

    protected

    • coalesceDescendants() — Recursively coalesces direct child information into full descendant information.
    • setDefaults() — Used to set up files that we want to exclude from parsing for performance reasons.

    public

    • __construct() — Constructs and initialises a new class manifest, either loading the data from the cache or re-scanning for classes.
    • getClassNames() — Returns a lowercase array of all the class names in the manifest.
    • getClasses() — Returns a map of lowercased class names to file paths.
    • getConfigs() — Returns an array of paths to module config files.
    • getDescendants() — Returns an array of all the descendant data.
    • getDescendantsOf() — Returns an array containing all the descendants (direct and indirect) of a class.
    • getImplementors() — Returns a map of lowercased interface names to the classes the implement them.
    • getImplementorsOf() — Returns an array containing the class names that implement a certain interface.
    • getInterfaces() — Returns a map of lowercased interface names to file locations.
    • getItemPath() — Returns the file path to a class or interface if it exists in the manifest.
    • getModules() — Returns an array of module names mapped to their paths.
