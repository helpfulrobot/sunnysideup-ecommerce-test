\SS_Query

Abstract query-result class.

Once again, this should be subclassed by an actual database implementation
such as .
It will only ever be constructed by a subclass of and
contain the result of a database query as an iteratable object.
Primarily, the SS_Query class takes care of the iterator plumbing, letting
the subclasses focusing on providing the specific data-access methods that
are required: , and

    Methods

    public

    • column() — Return an array containing all the values from a specific column. If no column is set, then the first will be returned.
    • current() — Iterator function implementation. Return the current item of the iterator.
    • first() — Iterator function implementation. Return the first item of this iterator.
    • key() — Iterator function implementation. Return the row number of the current item.
    • keyedColumn() — Return an array containing all values in the leftmost column, where the keys are the same as the values.
    • map() — Return a map from the first column to the second column.
    • next() — Iterator function implementation. Return the next record in the iterator.
    • nextRecord() — Return the next record in the query result.
    • numRecords() — Return the total number of items in the query result.
    • record() — Returns the next record in the iterator.
    • rewind() — Iterator function implementation. Rewind the iterator to the first item and return it.
    • seek() — Go to a specific row number in the query result and return the record.
    • table() — Return an HTML table containing the full result-set.
    • valid() — Iterator function implementation. Check if the iterator is pointing to a valid item.
    • value() — Returns the first column of the first record.
