Controller::handleRequest

Executes this controller, and return an {@link SS_HTTPResponse} object with the result.

This method first does a few set-up activities: - Push this controller ont to the controller stack - see for information about this. - Call - Defer to to determine which action should be executed Note: $requestParams['executeForm'] support was removed, make the following change in your URLs: "/?executeForm=FooBar" -> "/FooBar" Also make sure "FooBar" is in the $allowed_actions of your controller class. Note: You should rarely need to overload run() - this kind of change is only really appropriate for things like nested controllers - and are two examples here. If you want to make more orthodox functionality, it's better to overload or . Important: If you are going to overload handleRequest, make sure that you start the method with $this->pushCurrent() and end the method with $this->popCurrent(). Failure to do this will create weird session errors.
