    Methods

    public

    • SimpleHtmlLexer() — Sets up the lexer with case insensitive matching and adds the HTML handlers.
    • _addAttributeTokens() — Matches attributes that are either single quoted, double quoted or unquoted.
    • _addInTagTokens() — Pattern matches to parse the inside of a tag including the attributes and their quoting.
    • _addSkipping() — The lexer has to skip certain sections such as server code, client code and styles.
    • _addTag() — Pattern matches to start and end a tag.
    • _getParsedTags() — List of parsed tags. Others are ignored.

    Inherited from SimpleLexer

    public

    • SimpleLexer() — Sets up the lexer in case insensitive matching by default.
    • _decodeSpecial() — Strips the magic underscore marking single token modes.
    • _dispatchTokens() — Sends the matched token and any leading unmatched text to the parser changing the lexer to a new mode if one is listed.
    • _invokeParser() — Calls the parser method named after the current mode. Empty content will be ignored. The lexer has a parser handler for each mode in the lexer.
    • _isModeEnd() — Tests to see if the new mode is actually to leave the current mode and pop an item from the matching mode stack.
    • _isSpecialMode() — Test to see if the mode is one where this mode is entered for this token only and automatically leaves immediately afterwoods.
    • _reduce() — Tries to match a chunk of text and if successful removes the recognised chunk and any leading unparsed data. Empty strings will not be matched.
    • addEntryPattern() — Adds a pattern that will enter a new parsing mode. Useful for entering parenthesis, strings, tags, etc.
    • addExitPattern() — Adds a pattern that will exit the current mode and re-enter the previous one.
    • addPattern() — Adds a token search pattern for a particular parsing mode. The pattern does not change the current mode.
    • addSpecialPattern() — Adds a pattern that has a special mode. Acts as an entry and exit pattern in one go, effectively calling a special parser handler for this token only.
    • mapHandler() — Adds a mapping from a mode to another handler.
    • parse() — Splits the page text into tokens. Will fail if the handlers report an error or if no content is consumed. If successful then each unparsed and parsed token invokes a call to the held listener.
