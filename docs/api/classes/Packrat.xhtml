\Packrat

By inheriting from Packrat instead of Parser, the parser will run in linear time (instead of exponential like Parser), but will require a lot more memory, since every match-attempt at every position is memorised.

We now use a string as a byte-array to store position information rather than a straight array for memory reasons. This
means there is a (roughly) 8MB limit on the size of the string we can parse

  • Author: Hamish Friedlander
