Director::direct

Process the given URL, creating the appropriate controller and executing it.

Request processing is handled as follows: - Director::direct() creates a new SS_HTTPResponse object and passes this to Director::handleRequest(). - Director::handleRequest($request) checks each of the Director rules and identifies a controller to handle this request. - Controller::handleRequest($request) is then called. This will find a rule to handle the URL, and call the rule handling method. - RequestHandler::handleRequest($request) is recursively called whenever a rule handling method returns a RequestHandler object. In addition to request processing, Director will manage the session, and perform the output of the actual response to the browser.
