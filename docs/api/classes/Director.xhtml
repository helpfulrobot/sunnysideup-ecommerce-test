\Director

Director is responsible for processing URLs, and providing environment information.

The most important part of director is , which is passed a URL and will execute the
appropriate controller.
Director also has a number of static methods that provide information about the environment, such as
.

    Methods

    protected

    • force_redirect() — Skip any further processing and immediately respond with a redirect to the passed URL.
    • handleRequest() — Handle an HTTP request, defined with a SS_HTTPRequest object.
    • session_environment() — Check or update any temporary environment specified in the session

    public

    • absoluteBaseURL() — Returns the Absolute URL of the site root.
    • absoluteBaseURLWithAuth() — Returns the Absolute URL of the site root, embedding the current basic-auth credentials into the URL.
    • absoluteURL() — Turns the given URL into an absolute URL.
    • addRules() — Add URL matching rules to the Director.
    • baseFolder() — Returns the root filesystem folder for the site.
    • baseURL() — Returns the root URL for the site.
    • direct() — Process the given URL, creating the appropriate controller and executing it.
    • extract_request_headers() — Takes a $_SERVER data array and extracts HTTP request headers.
    • fileExists() — Returns true if the given file exists.
    • forceSSL() — Force the site to run on SSL.
    • forceWWW() — Force a redirect to a domain starting with "www."
    • getAbsFile() — Given a filesystem reference relative to the site root, return the full file-system path.
    • get_current_page() — Return the {@link SiteTree} object that is currently being viewed. If there is no SiteTree object to return, then this will return the current controller.
    • get_environment_type() — Can also be checked with {@link Director::isDev()}, {@link Director::isTest()}, and {@link Director::isLive()}.
    • get_template_global_variables()
    • isDev() — This function will return true if the site is in a development environment.
    • isLive()
    • isTest() — This function will return true if the site is in a test environment.
    • is_absolute() — Returns true if a given path is absolute. Works under both *nix and windows systems
    • is_absolute_url() — Checks if a given URL is absolute (e.g. starts with 'http://' etc.).
    • is_ajax() — Checks if the current HTTP-Request is an "Ajax-Request" by checking for a custom header set by jQuery or wether a manually set request-parameter 'ajax' is present.
    • is_cli() — Returns true if this script is being run from the command line rather than the webserver.
    • is_https() — Return whether the site is running as under HTTPS.
    • is_relative_url() — Checks if a given URL is relative by checking {@link is_absolute_url()}.
    • is_site_url() — Checks if the given URL is belonging to this "site" (not an external link).
    • makeRelative() — Turns an absolute URL or folder into one that's relative to the root of the site. This is useful when turning a URL into a filesystem reference, or vice versa.
    • protocol() — Return the current protocol that the site is running under.
    • protocolAndHost() — Returns the part of the URL, 'http://www.mysite.com'.
    • setBaseFolder() — Sets the root folder for the website.
    • setBaseURL() — Sets the root URL for the website.
    • setUrlParams() — Set url parameters (should only be called internally by RequestHandler->handleRequest()).
    • set_current_page() — Set the currently active {@link SiteTree} object that is being used to respond to the request.
    • set_environment_type() — Set the environment type of the current site.
    • test() — Test a URL request, returning a response object.
