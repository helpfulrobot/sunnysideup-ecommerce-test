Debug::friendlyError

Render a user-facing error page, using the default HTML error template rendered by {@link ErrorPage} if it exists. Doesn't use the standard {@link SS_HTTPResponse} class the keep dependencies minimal.

    Signature

    public function friendlyError([integer $statusCode = 500, [string $friendlyErrorMessage = null, [string $friendlyErrorDetail = null]]] )

    Parameters

    $statusCode — integer
    HTTP Status Code (Default: 500)
    $friendlyErrorMessage — string
    User-focused error message. Should not contain code pointers
    or "tech-speak". Used in the HTTP Header and ajax responses.
    $friendlyErrorDetail — string
    Detailed user-focused message. Is just used if no {@link ErrorPage} is found
    for this specific status code.

    Returns

    string
    HTML error message for non-ajax requests, plaintext for ajax-request.
